Работал над новым шаблоном в который подходил любой экспорт.

С помощью этого запроса создал шаблон:

```text
Создай мне один HTML-файл pricing_table.html — универсальный дизайн прайс-таблицы, который работает с экспортом из NetLicensing.
Как должно работать:

В файле есть два маркера-комментария:
html<!-- BEGIN:PricingTable -->
<!-- END:PricingTable -->
Между ними я вставляю HTML-экспорт из NetLicensing как есть, без изменений. При открытии файла в браузере скрипт автоматически парсит этот экспорт и отрисовывает красивую прайс-таблицу в фиксированном дизайне. Дизайн никогда не меняется — меняются только данные из экспорта.
Требования к дизайну (карточки в стиле SaaS):

Светлый фон #f5f6fa, белые карточки, акцент #4f46e5
Карточки в сетке (количество колонок = количество планов в экспорте, адаптивно)
Средняя карточка (план PRO) — выделена рамкой акцентного цвета и бейджем "Most popular"
Каждая карточка: название плана, описание, цена (с символом валюты €/$), список фич (✓ включено / – выключено, цветом) и квоты (значение -1 → "Unlimited"), кнопка CTA
Hover-эффект на карточках (поднятие + тень), адаптив для мобильных

Технические требования:

Блок с экспортом обёрнут в <div id="pricingExportSource" hidden> — не виден на странице
Скрипт запускается через DOMContentLoaded (чтобы находил блок независимо от его позиции в файле)
Парсер ищет элементы по подстроке класса ([class*="plan-column"], [class*="sku-feature"], [class*="sku-quota"] и т.д.) — работает с любым экспортом, не только NetLicensing
Дубли планов (вложенные .plan-column внутри .plan-column) устраняются — оставляется только внешний элемент
Файл полностью автономный, без внешних зависимостей
```
